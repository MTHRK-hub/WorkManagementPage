作成日: 2026-04-14

## 1. 現状分析

### 1-1. ファイル構成

現在、PC版とスマホ版で完全に別ファイルが存在している。

| 種別 | PC版 | スマホ版 |
|------|------|----------|
| メインページ | `index.html` | `index_phone.html` |
| メニュー | `src/html/00Menu.html` | `src/html/00Menu_phone.html` |
| コンテンツ (01〜08) | `src/html/0x***.html` (8ファイル) | `src/html/0x***_phone.html` (8ファイル) |
| メインCSS | `src/css/01Main.css` | `src/css/01Main_phone.css` |
| メニューCSS | `src/css/02Menu.css` | `src/css/02Menu_phone.css` |

**合計 20 HTML + 4 CSS = 24ファイル** → 統合後は **11 HTML + 2 CSS = 13ファイル** に削減。

### 1-2. 差分の分類

差分を3パターンに分類する。

#### パターンA: CSSのみ異なる（HTMLの内容は完全一致）
コンテンツ8ページ（01〜08）は `<link>` のCSS参照先だけが違い、本文は同一。

```
01JikoKanri.html      ← css/02Menu.css を参照
01JikoKanri_phone.html ← css/02Menu_phone.css を参照
（本文は完全に同じ）
```

#### パターンB: HTMLレイアウト構造が異なる
`00Menu.html` はPC版が2カラムの `<table>` 、スマホ版が1カラムの `<details>` 縦並び。
また各 `<iframe>` が指す内側HTMLのファイル名も `_phone` サフィックスで異なる。

#### パターンC: HTMLレイアウトと参照先が異なる
`index.html` はPC版が `<table>` で横並び3セル、スマホ版が縦並びの平置き構造。
参照するCSS・iframeのsrcも異なる。

---

## 2. 統合方針

**CSS Media Query を使いレスポンシブ対応に一本化する。**

- ブレークポイント: `max-width: 900px`（PC版 `1380px` 固定 ↔ スマホ版 制限なし の中間）
- PC版のファイル名をそのまま残し、スマホ版（`_phone`）を削除する
- `<meta name="viewport">` を追加してスマホ表示を正しく機能させる

## 4. HTML 統合設計

### 4-1. コンテンツページ（01〜08）— パターンA

**対象**: `01JikoKanri` 〜 `08OnkyoKanren` の各ペア

**変更内容**:
- `_phone.html` を削除
- PC版 `0x***.html` の `<link>` を統合後の `02Menu.css`（1ファイル）に統一
- `<meta name="viewport">` を追加

```html
<!-- 変更前 -->
<link href="../css/02Menu.css" rel="stylesheet" />

<!-- 変更後（追加のみ） -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<link href="../css/02Menu.css" rel="stylesheet" />
```

### 4-2. `00Menu.html` — パターンB

**課題**: PC版（2カラムtable）とスマホ版（1カラム縦並び）で構造が違う。

**対応方針**:  
HTML は PC版の2カラム `<table>` ベースを残しつつ、CSS で `max-width:900px` 時に `table/tr/td` を `display:block` にして1カラムに強制リフロー。
各 `<iframe>` の `src` は統合後のコンテンツページ（`_phone` 無し）に統一。

```css
/* 02Menu.css に追加 */
@media (max-width: 900px) {
  table { display: block; }
  tr    { display: block; }
  td    { display: block; width: 100%; }
}
```

`<iframe>` の src 変更例:
```html
<!-- 変更前 -->
<iframe class="if1" src="01JikoKanri.html">

<!-- 変更後（_phone 版は削除されるため、PC版 URL に統一） -->
<iframe class="if1" src="01JikoKanri.html">  <!-- 変更なし -->
```

スマホ版のみ参照していた `_phone.html` の URL は全て PC版のファイル名に自動的に統一される。

### 4-3. `index.html` — パターンC

**課題**:
- PC版は `<table>` でボタンとMENUラベルを横並び
- スマホ版は縦並び（tableなし）
- PC版には `<link rel="icon">` がある

**対応方針**:  
PC版の `<table>` を残し、CSS の Media Query で `display:block` に切り替えて縦並びにする。
`index_phone.html` は削除。

**変更点まとめ**:
```html
<!-- 追加 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- CSS参照を統合後の1ファイルに -->
<link href="src/css/01Main.css" rel="stylesheet" />

<!-- iframe の参照先を統合後に変更 -->
<iframe src="src/html/00Menu.html"></iframe>  <!-- 変更なし -->
```

```css
/* 01Main.css に追加 */
@media (max-width: 900px) {
  table { display: block; }
  tr    { display: block; }
  td    { display: block; }
}
```
